# analysis/ ‚Äî map

Cross-run aggregation + reporting for this study. Producers pull per-run
results from **W&B** (written by the wrapper's `post_training_analysis`; keys
live in `wandb_keys.py`), aggregate across cells (D √ó seed √ó subject), and
render the curated results in `reports/`. Run everything via the study
`Makefile` (`make all`), not individual scripts. See
`../../../docs/posthoc-analysis.md` for the conventions.

## What lives here

- **`reports/`** ‚Äî the curated results. One file per result (`r1‚Ä¶r9`) plus
  `INDEX.md`. **Start here.** Each `r*.md` has frontmatter (which W&B groups /
  script / data fed it) and `BEGIN/END` markers around script-regenerated regions.
- **`<producer>.py`** ‚Äî one script per result block (`nxd_scaling.py`,
  `rl_baseline.py`, `generative_match.py`, `mature_fewshot_curve.py`,
  `bootstrap_scaling.py`, `build_report.py`). Each writes its `<name>.json`
  (carrying a `_meta` provenance block) and its `fig_<slug>.png`.
- **`<producer>_verdict.md`** ‚Äî extended detail (fit stats, caveats) a report
  links to; regenerated by the matching producer where one exists.
- **`_meta.py`** ‚Äî builds the `_meta` block stamped into every result JSON
  (produced-at, dispatcher + wrapper git sha, W&B groups).
- **`wandb_keys.py`** ‚Äî the single source of truth for the W&B summary keys read
  from the wrapper (the producer‚Üîconsumer contract). Review this when bumping the
  wrapper pin in `../environment.lock`.
- **`provenance/`** ‚Äî launch breadcrumbs (not analysis outputs): `*_launched.json`
  = Beaker exp IDs + W&B groups + notes per arm; `heldout_rerun_validated.txt` =
  pipeline validation note; `wandb_report_url.txt` = the W&B report link. These
  trace a number in a report back to the exact runs/wrapper commit that made it.

## Not here

- Study cover / TL;DR / verdict / status ‚Üí the study `../README.md`.
- Cache/scratch (`_cache_*.json`, per-subject pulls, W&B download dirs) ‚Üí
  gitignored, never committed (see `../.gitignore`).
